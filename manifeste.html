Pourquoi vos outils devraient être aveugles

Internet devait être un espace de liberté. Quelque part en chemin, nous avons accepté que chaque fichier, chaque message, chaque clic soit visible par quelqu'un d'autre.

Le problème

Quand vous envoyez un fichier via un service de partage classique, vos données passent par leurs serveurs. Ces entreprises peuvent les lire, les analyser, les stocker. Parfois pour améliorer leurs services. Parfois parce que la loi les y oblige. Parfois parce qu'un employé mal intentionné a accès aux systèmes.

Le problème n'est pas que ces services soient mal conçus. Le problème est leur architecture même. Quand vos données passent par un serveur, quelqu'un d'autre que vous peut y accéder.

Les fuites de données ne sont pas des accidents rares. Elles sont la conséquence logique d'un modèle où la confiance est externalisée vers des tiers. Chaque serveur est une surface d'attaque. Chaque base de données est une cible potentielle.

Notre approche

VoidLab prend le chemin inverse. Au lieu de vous demander de nous faire confiance, nous avons construit des outils qui n'ont pas besoin de votre confiance.

Transfer, notre premier outil, envoie vos fichiers directement d'un appareil à l'autre. Aucun serveur intermédiaire ne voit passer vos données. Même nous, les créateurs de l'outil, ne pouvons pas savoir ce que vous envoyez, à qui, ni quand.

Ce n'est pas une promesse marketing. C'est une propriété mathématique de l'architecture que nous utilisons.

Zéro confiance requise

Vous n'avez pas à nous croire sur parole. Le code est ouvert, les protocoles sont documentés, les implémentations sont vérifiables.

Zéro connaissance

Nous ne savons rien de ce que vous faites. Pas de compte, pas d'identifiant, pas de métadonnées. Vos fichiers sont protégés de bout en bout.

Zéro trace

Pas de logs, pas d'historique, pas de persistance. Les clés sont détruites après chaque session. Ce qui n'existe pas ne peut pas fuiter.

Ce que nous ne faisons pas

Nous ne collectons pas vos données. Nous n'avons pas de trackers, pas de cookies analytiques, pas de pixels de suivi. Nous ne vendons rien à personne parce que nous n'avons rien à vendre.

Nous ne monétisons pas vos usages. VoidLab est gratuit et le restera. Notre modèle repose sur la conviction que des outils de base pour la vie privée ne devraient pas être un luxe.

Nous ne faisons pas de compromis sur la sécurité. Si une fonctionnalité implique de collecter des données ou de réduire la protection, nous ne l'implémentons pas.

Ce que nous ne pouvons pas promettre

Protection de l'adresse IP. Les connexions directes entre appareils révèlent les adresses IP des participants. Si l'anonymat total est important pour vous, utilisez VoidLab via Tor Browser ou un VPN qui protège contre les fuites WebRTC.

Audit professionnel. Notre code est ouvert mais n'a pas été audité par une firme de sécurité externe. Nous encourageons les chercheurs à examiner notre implémentation.

Disponibilité garantie. VoidLab est un projet indépendant. Nous faisons de notre mieux mais ne pouvons pas offrir les garanties d'une infrastructure d'entreprise.

Transparence totale

Tout ce que nous construisons est visible sur GitHub. Vous pouvez lire chaque ligne de code, vérifier chaque algorithme, auditer chaque décision technique.

Si vous trouvez une faille, dites-le nous. Si vous avez des suggestions, elles sont les bienvenues. VoidLab appartient à ceux qui l'utilisent.
